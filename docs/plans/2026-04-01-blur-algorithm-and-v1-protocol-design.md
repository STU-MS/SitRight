# 模糊算法与第一版串口协议设计

- 日期：2026-04-01
- 范围：SitRight 第一版联调协议收敛与 blurLevel 非线性映射设计
- 目标：
  - 统一设备到上位机运行态协议为 `<number>\n`
  - 降低硬件与软件联调成本，避免协议语义分歧
  - 构建符合体感的非线性模糊反馈链路

## 1. 已确认决策

1. 协议输出策略：方案3
- 运行态使用纯数字行协议。
- 调试信息仅在编译开关开启时输出。

2. 设备输出语义：方案3
- 支持输出角度或 blurLevel，通过编译开关切换。
- 第一版默认输出 blurLevel。

3. 上位机解析策略：方案1
- 严格模式，仅接受纯数字行。
- 非数字行直接丢弃并计数。

## 2. 方案对比与选型

评估路径：

1. 路径A（推荐）：设备内完成角度到 blurLevel 映射
- 优点：协议最简、联调最快、职责边界清晰。
- 缺点：调参需改固件常量。

2. 路径B：设备输出角度，上位机映射 blurLevel
- 优点：上位机调参灵活。
- 缺点：联调语义跨端，复杂度更高。

3. 路径C：双通路并存
- 优点：灵活过渡。
- 缺点：状态组合和测试成本显著增加。

最终采用路径A，配合必要编译开关保留实验能力。

## 3. 总体架构与职责边界

1. 设备采集层
- 读取 MPU6050 并计算姿态角。

2. 设备评分层
- 基于校准参数将角度转换为 0~100 的 blurLevel。

3. 设备通信层
- 运行态仅输出 `<number>\n`。
- 调试输出受 `DEBUG_SERIAL` 控制，默认关闭。

4. 上位机协议层
- 严格解析纯数字行并做范围校验。

5. 上位机控制层
- 对合法 level 做平滑和状态管理（Receiving/Timeout）。

6. 上位机视觉层
- 依据非线性映射输出遮罩、暗化和提示透明度。

## 4. 设备端 blurLevel 算法设计

### 4.1 偏差归一化

设当前角度为 `angle`，坐正基准为 `normal`，驼背参考为 `slouch`。

- 绝对偏差：
  - `d = |angle - normal|`
- 参考跨度：
  - `r = max(|slouch - normal|, rMin)`
  - 建议 `rMin = 5°`（避免分母过小）
- 归一化输入：
  - `x = clamp(d / r, 0, 1.2)`

### 4.2 分段非线性压迫曲线

为满足“0~30 柔和、30~70 明显、70~100 快速压迫”，采用分段幂函数：

- `x <= 0.3`:
  - `b = 30 * (x / 0.3)^1.6`
- `0.3 < x <= 0.7`:
  - `b = 30 + 40 * ((x - 0.3) / 0.4)^1.2`
- `x > 0.7`:
  - `b = 70 + 30 * ((x - 0.7) / 0.5)^0.8`

最终输出：
- `blurLevel = round(clamp(b, 0, 100))`

### 4.3 轻量稳定化

设备端可选最小稳定化以降低抖动：
- 3点中值滤波，或
- 一阶平滑：`y_t = y_{t-1} + alpha * (blur_t - y_{t-1})`，建议 `alpha = 0.25`

## 5. 第一版串口协议与开关约束

串口协议为**双通道**设计：运行态数据和校准回包共享同一串口，上位机通过行首前缀区分。

### 5.1 运行态协议

固定：

```txt
<number>\n
```

示例：

```txt
0
37
100
```

### 5.2 校准回包协议

校准回包以 `ACK:` 或 `ERR:` 开头，始终输出，不受编译开关控制：

```txt
ACK:SET_NORMAL,ANGLE:12.34\n
ACK:SET_SLOUCH,ANGLE:25.67\n
ERR:BUSY\n
ERR:UNKNOWN_CMD\n
```

详见 `docs/plans/2026-04-01-serial-calibration-design.md`。

### 5.3 编译开关建议

- `OUTPUT_MODE_BLUR`（默认）
- `OUTPUT_MODE_ANGLE`（实验）
- `DEBUG_SERIAL`（默认关闭）

约束：
- 当 `DEBUG_SERIAL=0` 时，禁止任何调试文本输出。
- ACK/ERR 回包不属于调试文本，不受此开关影响。

## 6. 上位机严格解析与容错

1. 严格解析
- 仅接受整数字符串并换行分帧。
- blur 模式范围校验 `[0,100]`。

2. 非法输入处理
- 非法行直接丢弃。
- `ParseErrorCount++`，用于主窗口可观测性。

3. 超时降级
- 超过 2000ms 未收到合法值，状态转 `Timeout`。
- Overlay 目标强度逐步回退，避免高惩罚残留。

## 7. 上位机非线性视觉映射

设 `n = level / 100`，并对结果钳制到 `[0,1]`：

- `MaskOpacity = 0.05 + n^1.4 * 0.65`
- `DarkenOpacity = n^1.8 * 0.25`
- `MessageOpacity = level > 25 ? min(1, (level - 25)/40) : 0`

文案等级建议：
- 0~20：保持坐姿
- 21~50：请微调坐姿
- 51~80：请立即调整坐姿
- 81~100：坐姿偏差过大

## 8. 验收标准

1. 协议一致性
- 运行态串口抓包仅出现纯数字行。

2. 解析稳定性
- 注入混合噪声文本时，上位机不崩溃，非法行被丢弃并计数。

3. 超时行为
- 断开设备后 2 秒内进入 `Timeout`，Overlay 强度可见回退。

4. 映射体感
- 低区间变化柔和、中区间明显、高区间快速压迫，符合设计目标。

5. 回归
- 打开调试开关时日志可见，关闭后协议流不被污染。

## 9. 风险与后续

1. 风险
- 双输出语义（angle/blur）可能造成配置错配。

2. 缓解
- 上位机配置中显式标注输入语义，默认锁定 blur。
- 在主窗口展示当前解析模式与最近合法值。

3. 第二版可选增强
- 自适应协议探测模式。
- 运行时下发参数调优（阈值、指数、平滑系数）。

## 10. 结论

第一版采用“设备输出 blurLevel + 上位机严格纯数字解析 + 上位机非线性视觉映射”的最小闭环，能够在保持扩展能力的同时显著降低联调成本并提升演示稳定性。
